# Cloche Docker Base Image Design

**Date:** 2026-03-06
**Status:** Design

## Problem

The default Dockerfile generated by `cloche init` builds `cloche-agent` from the cloche
Go source inside the image. This only works for the cloche project itself (which has the
source available). For any other project, the template is broken out of the box.

The root `Dockerfile` (used by the cloche project's own `.cloche/`) makes the same
mistake, and is also unnecessarily large (based on `golang:1.25` for the final stage).

## Solution

A `cloche-base` image: a minimal Debian-slim image with `git` and `cloche-agent`
pre-installed. Projects start from this image and add their own tooling on top.

Using `cloche-base` is **not required** — projects may use any Dockerfile they like.

---

## `cloche-base` image contents

Built from `docker/cloche-base/Dockerfile` in the cloche repo.

| Component | Why |
|-----------|-----|
| `debian:12-slim` | Small, stable, widely compatible |
| `git` | Required by `cloche-agent` for branch/commit operations |
| `ca-certificates` | TLS for any outbound requests agents make |
| `cloche-agent` binary | The in-container workflow executor |
| `agent` user | Unprivileged user; workdir `/workspace` owned by `agent` |

**Not included:** Node.js, npm, claude-code, language runtimes, build tools. These are
project-specific and belong in the project's own Dockerfile.

### `docker/cloche-base/Dockerfile`

```dockerfile
FROM golang:1.25 AS builder
WORKDIR /src
COPY go.mod go.sum ./
RUN go mod download
COPY . .
RUN CGO_ENABLED=0 go build -o /cloche-agent ./cmd/cloche-agent

FROM debian:12-slim
RUN apt-get update \
    && apt-get install -y --no-install-recommends git ca-certificates \
    && rm -rf /var/lib/apt/lists/*
COPY --from=builder /cloche-agent /usr/local/bin/cloche-agent
RUN useradd -m -s /bin/bash agent
WORKDIR /workspace
RUN chown agent:agent /workspace
USER agent
```

Self-contained multi-stage build. No pre-built binary required on the host.

## Tags

Both tags are applied on every build:

- `cloche-base:latest`
- `cloche-base:<version>` — derived from `git describe --tags --always`

Local only for now; registry publishing deferred.

---

## Makefile changes

```makefile
VERSION ?= $(shell git describe --tags --always 2>/dev/null || echo "dev")

docker-base:
	docker build -f docker/cloche-base/Dockerfile \
		-t cloche-base:latest \
		-t cloche-base:$(VERSION) \
		.

docker-build: docker-base
	docker build -t cloche-agent:latest -f .cloche/Dockerfile .

install: build docker-build
	...  (unchanged beyond this)
```

`docker-build` now depends on `docker-base` so the cloche project image is always built
on top of a fresh base.

---

## Root `Dockerfile` update (cloche project's own image)

The root `Dockerfile` (`.cloche/Dockerfile` for the cloche project) is simplified to
build on top of `cloche-base:latest`:

```dockerfile
FROM cloche-base:latest
USER root
RUN apt-get update \
    && apt-get install -y --no-install-recommends nodejs npm \
    && rm -rf /var/lib/apt/lists/*
RUN npm install -g @anthropic-ai/claude-code
USER agent
```

The golang builder stage is removed — `cloche-agent` now comes from the base.

---

## Default Dockerfile template (from `cloche init`)

The template in `cmd/cloche/init.go` is replaced with:

```dockerfile
FROM cloche-base:latest
USER root

# Add your project's dependencies here.
# Example:
#   RUN apt-get update && apt-get install -y --no-install-recommends python3 && rm -rf /var/lib/apt/lists/*
#   RUN npm install -g @anthropic-ai/claude-code

USER agent
```

The `--image` flag to `cloche init` is repurposed as `--base-image` (default:
`cloche-base:latest`) to allow pinning to a specific version:

```
cloche init --base-image cloche-base:v1.2.0
```

This replaces the `FROM cloche-base:latest` line in the generated Dockerfile.

## Changes required

| File | Change |
|------|--------|
| `docker/cloche-base/Dockerfile` | New file — base image definition |
| `Makefile` | Add `docker-base` target; make `docker-build` depend on it; add `VERSION` |
| `Dockerfile` (root / cloche project) | Simplify to `FROM cloche-base:latest` + add claude-code |
| `.cloche/Dockerfile` (cloche project) | Same simplification as root Dockerfile |
| `cmd/cloche/init.go` | New template using `cloche-base:latest`; rename `--image` to `--base-image` |
